## 2024-05-24 - [UX improvement] Battery Charging Pulse and System Broadcast Icons
**Learning:** Android 14+ (API 34) broadcast receivers for system-level protected intents like `ACTION_CONNECTION_STATE_CHANGED` and `NETWORK_STATE_CHANGED_ACTION` must use `Context.RECEIVER_EXPORTED` when registered dynamically in SystemUI, contrary to intuition about spoofing protection.
**Action:** Always use `RECEIVER_EXPORTED` conditionally for these intents on Tiramisu and above to prevent `SecurityException` crashes in the SystemUI hook. Furthermore, vector drawable assets must use a default fill color of `#FFFFFF` to allow Jetpack Compose's `tint` modifier to dynamically colorize them without graphical artifacts.
## 2024-05-24 - [UX improvement] Dashboard & CI Fixes
**Learning:** Jetpack Compose does not support `try-catch` blocks directly around composable invocations (e.g., `Image()`) if they emit UI. Wrap the underlying data-fetching logic in a `try-catch` block first, and conditionally emit the composable based on success. When calling permission-restricted APIs (like `BluetoothAdapter`) within Xposed hooks where standard Android permission flows aren't strictly applicable, use `@SuppressLint("MissingPermission")` to resolve lint compilation errors.
**Action:** Always refactor data-fetching logic outside of composable calls and handle null states. Use `@SuppressLint("MissingPermission")` when necessary in `IslandController.kt` to pass CI lint checks.
## 2024-05-24 - [Architectural Rewrite] LiveActivityModel & Gesture Engine
**Learning:** For overlay layouts dynamically scaling sizes in WindowManager, use `WRAP_CONTENT` for `WindowManager.LayoutParams` with `FLAG_NOT_TOUCH_MODAL` (so background touches pass through) and `Modifier.animateContentSize()` in Compose to ensure children components like Dashboards do not clip. To avoid Jetpack Compose gesture conflicts (e.g., `detectTapGestures` vs `detectDragGestures`) in complex overlays, use a custom `awaitPointerEventScope` with `awaitEachGesture` and `awaitFirstDown` to precisely track coordinates and time elapsed to route single/double/triple taps and directional swipes. Polling hardware sensors (like CPU and battery temperatures via Linux `sysfs` files) must be done asynchronously using a Coroutine `Flow` bound to `Dispatchers.IO` to avoid blocking the SystemUI main thread.
**Action:** Implement a `sealed class` for `LiveActivityModel`. Use Coroutine Jobs (`job?.cancel()`) for transient notification timers. Declare a stub `NotificationListenerService` in `AndroidManifest.xml` to prevent `SecurityException` when hooking media sessions. Add `FLAG_ACTIVITY_NEW_TASK` to intents launching from SystemUI. Use `awaitPointerEventScope` for all Island gesture routing.
## 2024-05-24 - [Architectural Rewrite] Jetpack Compose Lifecycle Fixes for SystemUI Overlays
**Learning:** SystemUI boot loops are often caused by Jetpack Compose lacking a valid `ViewTreeLifecycleOwner` when injected via Xposed `WindowManager.LayoutParams`. Furthermore, using `WRAP_CONTENT` alongside `LAYOUT_IN_DISPLAY_CUTOUT_MODE_ALWAYS` can panic the Android layout engine on some custom ROMs.
**Action:** Implement a custom class (`RedwoodLifecycleOwner`) that inherits both `LifecycleOwner` and `SavedStateRegistryOwner`, explicitly trigger `onCreate`/`onStart`/`onResume`, and bind it to the `ComposeView` via extensions. Always use `MATCH_PARENT` for `WindowManager.LayoutParams` overlay bounds, relying on Compose for internal dimension limits. Temporarily quarantine async initialization loops (`init {}`) during deployment to safely isolate UI layer issues.
## 2024-05-24 - [Architectural Rewrite] Delayed Injection Safety
**Learning:** When developing SystemUI Xposed modules, fatal crashes during `WindowManager.addView` or Compose initialization cause immediate boot loops, forcing factory resets or complex recovery procedures.
**Action:** Wrap the primary UI injection logic in a delayed `Handler(Looper.getMainLooper()).postDelayed` block (e.g., 15 seconds). This acts as a safety kill-switch, allowing time for the device to boot and giving the developer a window to check logs or disable the module via LSPosed before the crash loop begins.
## 2024-05-24 - [CI/CD] GitHub Actions APK Signing
**Learning:** To generate a signed release APK via GitHub Actions without hardcoding secrets in Gradle files, you can decode a base64-encoded keystore from repository secrets (`${{ secrets.KEYSTORE }}`), run `./gradlew assembleRelease` to generate an unsigned APK, and manually invoke the Android SDK `apksigner` tool to sign it using the decoded keystore and password secrets.
**Action:** Overrode standard "do not modify workflow" rules to update `.github/workflows/android_build.yml` with explicit `apksigner` steps to fulfill the user's request for a signed release build.
## 2024-05-24 - [Architectural Rewrite] Advanced UI Defenses for SystemUI
**Learning:** Jetpack Compose components crash SystemUI if provided `NaN` or `Infinite` progress values. Use `.coerceIn(0f, 1f)`. Furthermore, external multi-platform Compose UI libraries (like custom sliders) often cause `IllegalArgumentException` crashes within SystemUI Xposed hooks because they expect standard `Activity` contexts. Rely on native Material 3 components. When implementing custom seeking logic or dragging interactions on sliders, use `interactionSource.collectIsDraggedAsState()` to temporarily detach the UI thumb from real-time playback position updates, preventing jitter until the drag finishes. Android strictly requires app updates to match the cryptographic signature of the originally installed APK. Installing a Release APK over a Debug-signed installation fails with `INSTALL_FAILED_UPDATE_INCOMPATIBLE`.
**Action:** Use native Compose Slider instead of external WavySlider. Always null-check division logic for progress bars. Use explicit `painterResource` with local XML vectors instead of `android.R.drawable` generic system icons to prevent OEM vector crashes. Always uninstall previous debug builds before flashing production signed APKs.
## 2024-05-24 - [Architectural Rewrite] Advanced Grabber and Settings Rework
